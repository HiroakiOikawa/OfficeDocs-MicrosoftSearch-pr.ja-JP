# <a name="make-content-easy-to-find"></a>簡単に見つけられるコンテンツ

Microsoft Search は、ユーザーが関連コンテンツを見つけるのに役立つ機能です。 これは、イントラネットと web コンテンツの両方を検索するための安全な方法です。 この種の web および組織間の統合は、Microsoft からのみ入手できます。 Microsoft Search では、管理者は組織の知識を利用して、ユーザーが関連性のあるコンテンツを簡単に見つけられるようにすることができます。 

## <a name="components-that-find-content"></a>コンテンツを検索するコンポーネント
Microsoft Search では、管理者が[ブックマーク](manage-bookmarks.md)、 [PowerApps](integrate-powerapps.md)、 [Q&A](manage-qas.md)、およびコンテンツを見つけやすい[場所](manage-locations.md)に作成します。 これらの各検索コンポーネントには、その検索コンポーネントをトリガーするタイトル、URL、キーワードのセットが含まれます。

## <a name="bookmarks"></a>ブックマーク
[ブックマーク](manage-bookmarks.md)は、いくつかの手順で作成できます。 各ブックマークには、そのブックマークをトリガーするタイトル、URL、キーワードのセットが含まれます。 ブックマークには複数のキーワードを含めることができ、複数のブックマークは同じキーワードを共有できます。 ただし、予約済みのキーワードは共有できません。 ブックマークを作成または変更すると、検索インデックスが更新され、ユーザーがすぐにブックマークを使用できるようになります。

組織で[SharePoint](http://sharepoint.com/)に**昇格**した結果がある場合は、それらの結果を Microsoft Search にインポートできます。 昇格した結果を使用すると、検索結果をすばやく設定し、ユーザーがコンテンツを使用できるようにしたり、設定後すぐに Microsoft 検索をより効果的にすることができます。 関連する検索結果の名前付け方法と作成方法を理解するには、SharePoint の昇格した結果を参照として使用することをお勧めします。 

### <a name="add-or-edit-bookmarks-by-using-browser-extensions"></a>ブラウザー拡張機能を使用してブックマークを追加または編集する
検索管理者は、ブラウザーの拡張機能を使用して検索コンテンツを簡単に作成できます。 サイトをブックマークとして追加するには、ブラウザー拡張機能をインストールします。 その後、サイトに移動し、ブックマークとして追加します。 詳細については、「[ブックマークを管理](manage-bookmarks.md)する」を参照してください。

現在、 [Microsoft Edge](https://www.microsoft.com/windows/microsoft-edge)および[Google Chrome](https://www.google.com)ではブラウザー拡張機能を使用できます。 
- エッジ拡張機能をダウンロードするには、 [Microsoft Store](https://www.microsoft.com/p/microsoft-search-content-creator/9nrqdbcbwq55?activetab=pivot:overviewtab)にアクセスしてください。
- Chrome 拡張機能をダウンロードするには、 [chrome web store](https://chrome.google.com/webstore/detail/microsoft-search-content/nocnablpaoeecfmfnjoheefkogmleipm)に移動します。

## <a name="powerapps"></a>PowerApps

既存の[PowerApps](integrate-powerapps.md)を[ブックマーク](manage-bookmarks.md)に追加することで、休暇時間の入力や経費の報告などの作業をユーザーが完了できるようになります。 

[PowerApps](integrate-powerapps.md)を使用すると、ブラウザーまたは携帯電話やタブレットで実行されるビジネスアプリを作成できます。 コーディングの手順は必要ありません。 PowerApps は任意のブラウザーおよび任意のデバイスで使用できます。 追加にかかる時間は1分未満です。 PowerApps の詳細については、次の記事を参照してください。
- [ガイド付き学習](https://docs.microsoft.com/learn/browse/?products=powerapps)
- [PowerApps ドキュメント](https://docs.microsoft.com/powerapps/maker/canvas-apps/get-sessionid)
- [PowerApps ホーム](https://make.preview.powerapps.com/environments/839eace6-59ab-4243-97ec-a5b8fcc104e4/home)

### <a name="add-a-powerapp-to-a-bookmark"></a>ブックマークに PowerApp を追加する

1. 追加する PowerApp の[アプリ ID](https://docs.microsoft.com/powerapps/maker/canvas-apps/get-sessionid#get-an-app-id)を検索します。
1. Microsoft 365[管理センター](https://admin.microsoft.com)で、[**microsoft Search**の**設定** > ] に移動します。 
1. ブックマークを追加するか、PowerApp を追加する既存のブックマークを検索します。
1. [**ブックマークの設定**] で、[ **Power App**] を選択します。 [ **Power App の追加**] を選択します。
1. **アプリ ID**を入力します。 高さと幅は自動的に調整されます。 [ブックマーク](manage-bookmarks.md)は縦方向と横方向の両方をサポートできますが、現在サイズを変更することはできません。 テストが容易になるように、ブックマークプレビューは完全に機能する PowerApp を示しています。
1. **[公開]** または **[下書きに保存]** を選択します。

## <a name="qa"></a>Q&A

[Q&を](manage-qas.md)作成することは、[ブックマーク](manage-bookmarks.md)の作成に似ています。 Q&A では、web ページへのリンクだけでなく、ユーザーに質問するための回答を提供することができます。 利用可能なツールを使用して、リッチテキスト形式の回答を書式設定できます。 ブックマークと Q&が同じキーワードを共有している場合は、ブックマークの結果が最初に表示されます。 ブックマークと同様に、q&A を追加または変更した直後に、Q&インデックスが更新されます。 

### <a name="supported-html-tags"></a>サポートされている HTML タグ

HTML コンテンツを使用するか、応答に HTML タグを追加するか、または説明を追加することができます。 次の HTML タグがサポートされています。
 
- blockquote
- div
- em
- h1、h2、h3、h4
- ol、ul、li
- p
- pre
- span
- strong
- table、thead、tbody、tr、th、td
- u
- a
- code
- br
- hr
- img

サポートされていないタグは、無視されるか、テキストとして表示されます。 カードを必ずプレビューするようにします。

> [!Note]
> テンプレートファイルにエラーがある場合は、Q&のアイテムをインポートすることはできません。 エラーが発生しないようにするには、インポートファイルが適切に書式設定されており、必要なすべての情報が含まれていることを確認してください。 [インポートエラーを防ぐ](#prevent-import-errors)方法については、「詳細」を参照してください。

## <a name="locations"></a>場所

[場所](manage-locations.md)によって、ユーザーはアドレスを検索し、組織の建物を見つけることができます。 **場所**機能を使用すると、オフィス、キャンパス、建物の正確な場所に加えて、指示やナビゲーションを行うことができます。 最良の結果を得るために、管理者は組織の重要な場所をすべて Microsoft Search に追加する必要があります。 [ブックマーク](manage-bookmarks.md)や[Q&A](manage-qas.md)とは異なり、場所のインデックスはすぐに更新されません。 新しいまたは変更された場所が検索結果に表示されるまで数時間かかる場合があります。

## <a name="get-started"></a>作業の開始
ユーザーが必要とする情報を見つけて、その情報を簡単に見つけられるようにするには、次の方法を試してみてください。

- イントラネットの検索ログを使用して、トラフィックが最も多いサイトやページを判別する。
- 毎日または毎週の頻度で使用されるアプリ、サイト、およびツールを判別する。
- 従業員の福利厚生の直接リンクを確認する。
- ユーザーが把握する必要があるポリシーとプロセスを確認する。
- ユーザーがサポートを担当するユーザーとその方法を決定します。
- Seasonally またはビジネスサイクルに基づいて、定期的に必要な情報を取得します。 例としては、休暇または四半期ごとの財務更新を予約するツールを探している人がいます。
- 地域ユーザーまたはモバイルユーザーのポリシーを収集します。 例としては、場所によって異なる利点があります。
- 一般的な web 検索の内部サイトと情報を決定します。 例としては、トラフィック、公共交通情報、地方天気、企業パートナーから入手できる割引、健康およびフィットネスプログラムなどがあります。
- 会社がスポンサーになっているイベント、会議、社員旅行などに関する情報を見つける。
- 代表的な IT、人事、サポート関連の問題やよく寄せられる質問 (FAQ) と回答について調べる。

## <a name="involve-smes-and-users"></a>Sme およびユーザーを参加させる
組織では、ユーザーは単純な複雑なトピックを検索します。 簡単な例として、office のアドレスと従業員の利点が挙げられます。 複雑な例としては、新しい作業プロセス、技術情報、および操作内容があります。 さまざまな種類のコンテンツを作成または検索するには、さまざまなフィールド、サブジェクト、およびテクノロジに関する専門知識が必要です。 

ほとんどの検索管理者には、すべてのサブジェクトに関する特定の知識がありません。 外部リソースからの支援なしで利用可能なコンテンツの量を拡大縮小するには、組織内の他のユーザーから専門知識と知識を求めます。

### <a name="get-content-from-smes"></a>Sme からコンテンツを取得する
人事、サポート、販売、テクノロジ、その他の主要な分野の専門家を含む、組織内の該当分野の専門家 (Sme) を活用します。 Microsoft 検索エディターとして追加すると、Sme はコンテンツを直接投稿できます。 

### <a name="involve-your-users"></a>ユーザーの参加をはかる
ユーザーにブックマークするリソースを提案してもらうようにします。 また、破損または無効なリンクなどのエラーを報告するようにユーザーに要求します。

## <a name="set-up-components"></a>コンポーネントをセットアップする
単一または一括の[ブックマーク](manage-bookmarks.md)、 [Q&A](manage-qas.md)、および[場所](manage-locations.md)を追加または編集するには、以下のセクションの手順を実行します。 

### <a name="add-or-edit-a-single-bookmark-qa-or-location-component"></a>単一のブックマーク、Q&A、または location コンポーネントを追加または編集する
1. Microsoft 365[管理センター](https://admin.microsoft.com)で、[**microsoft Search**の**設定** > ] に移動します。 コンポーネントの [名前] タブを選択します。既定では、[**ブックマーク**] タブが選択されています。
1. コンポーネントを追加するには、[**新規追加**] を選択します。 
1. コンポーネントを編集するには、関連するコンポーネントの一覧でブックマークを選択します。 
1. 情報を追加または編集すると、プレビューが自動的に更新されます。

### <a name="bulk-add-or-edit-components"></a>コンポーネントを一括で追加または編集する
**インポート**および**エクスポート**機能を使用すると、検索管理者は[ブックマーク](manage-bookmarks.md)、 [Q&A](manage-qas.md)、[場所](manage-locations.md)を一括作成または編集できます。 この機能は、管理者が多数のコンポーネントを追加または編集する場合に役立ちます。 

インポートおよびエクスポート機能は、次の機能を提供します。
- **一括追加**。 コンポーネントのテンプレートファイルに詳細を追加してからインポートします。
- **一括編集**。 CSV ファイルにコンポーネントをエクスポートし、エクスポートした CSV でブックマークの詳細を編集した後、更新された CSV をインポートします。
- ** [SharePoint](http://sharepoint.com/)から昇格したサイトをインポート**します。 この機能は[ブックマーク](manage-bookmarks.md)にのみ適用されます。
- **バックアップ**。 CSV ファイルにコンポーネントをエクスポートします。

コンポーネントをインポートまたはエクスポートするには、次の手順を実行します。
1. コンポーネントの [名前] タブの右上隅で、[**インポート**] を選択します。 
1. CSV ファイル内のすべての既存のコンポーネントをダウンロードするには、[**エクスポート**] を選択します。
1. 右側のウィンドウで、CSV ファイルを使用して、または[SharePoint](http://sharepoint.com/)からインポートするオプションを選択します。
1. 必要なフィールドと詳細の一覧を取得するには、コンポーネントのテンプレートファイルをダウンロードします。 
1. テンプレートファイルでコンポーネントの詳細を追加または編集します。 その後、コンピューターに保存します。 
1. コンポーネントの [**インポート**] ウィンドウで、[**参照**] を選択します。 次に、必要な CSV ファイルを選択して、[**インポート**] を選択します。

### <a name="template-guidelines"></a>テンプレートのガイドライン
テンプレートファイルを操作するときは、以下のガイドラインと制限に注意してください。
- これらのフィールドでは、 *Id*、 *Last Modified*、 *last modified*の各フィールドのデータを編集しないでください。
- 既存のブックマークの*Id*を指定した場合は、インポートファイルの情報に置き換えられます。
- 既存のファイルに同じタイトルまたは URL を持つブックマークがある場合、ブックマークはインポートファイルの情報で更新されます。
- テンプレートファイルのすべてのフィールドが必須であるわけではなく、ブックマークの状態によって必須のフィールドが異なる場合があります。
- [*状態*] フィールドに基づいて、ブックマークは**下書き**、**提案**済み、または**スケジュール**済みとして保存されます。 それ以外の場合は、自動的に公開されます。
- 複数の組織を管理する場合は、1つの組織からブックマークをエクスポートして、別の組織にインポートすることができます。 ただし、インポートする前に *ID* 列のデータを削除する必要があります。

> [!Note]
> テンプレートファイルにエラーがある場合、コンポーネントアイテムをインポートすることはできません。 エラーが発生しないようにするには、インポートファイルが適切に書式設定されており、必要なすべての情報が含まれていることを確認してください。

### <a name="prevent-import-errors"></a>インポート エラーを回避する

必要なデータが不足しているか無効な場合は、エラーメッセージが表示されます。 修正される行と列の詳細情報を含むログファイルが生成されます。 必要な編集を行い、ファイルを再度インポートします。 すべてのエラーが解決されるまで、ブックマークをインポートまたは保存することはできません。

エラーが発生しないようにするには、インポートファイルが適切に書式設定され、次の要件を満たしていることを確認してください。
- ヘッダー行と、インポートテンプレート内のすべての列が含まれています。
- 列の順序は、インポートテンプレートと同じです。
- すべての列には値があります。ただし、空の場合は、 *Id*、 *last Modified*、および*last modified*という3つの値があります。 
- [*状態*] 列が空ではありません。

### <a name="titles-and-descriptions"></a>タイトルと説明
接続されたタイトルと説明は、結果が検索クエリに応答するかどうかをユーザーが判断するのに役立ちます。 適切なタイトルと説明は、結果の中心となる目的を反映しています。 例としては、説明による**Childcare の利点**として、 *childcare 費用の支払いに役立つ利点について*説明します。 この接続データを使用すると、 **childcare**を検索するユーザーは、金銭サポートの利点を見つけ、リンクを取得して詳細を見つけることができます。

### <a name="keywords"></a>キーワード
キーワードを使用すると、組織内のユーザーは関連するコンテンツを検索して見つけることができます。 関連する検索結果にキーワード用語を関連付ける必要があります。 Microsoft Search では、コンテンツのタイトルと URL に基づいてキーワードが推奨されます。 その他のキーワードを特定するには、次の質問に対する回答を取得します。

1. **特定された情報を見つけられる検索語** 組織内の既存の用語、および関連するバリエーション、略語、主題、トピックを参照してください。
1. **ユーザーがこの情報について説明するためにどのようなバリエーションまたは単語を使用するか。** サポートチームにこれらのキーワードの提供を依頼してください。

たとえば、休暇要求を送信するためのツールにリンクする結果を作成した場合、[キーワード**休暇**] と [**休暇の提出] 要求**は、を含めるための適切なオプションです。 組織内のユーザーは、**休日****または休暇**の休暇関連情報を検索することもあります。 ユーザーが関連性のあるコンテンツを簡単に見つけられるようにするには、それらのキーワードを追加します。また、**休日要求の提出**や**要求時間**の終了などを追加します。

### <a name="reserved-keywords"></a>予約済みキーワード
 予約済みキーワードは、結果をトリガーする一意の用語またはフレーズです。 他のキーワードとは異なり、予約済みキーワードは1つの結果にのみ関連付けられます。 Microsoft Search が使用度に基づいて学習できるように、予約済みキーワードはあまり多用しないようにしてください。

例としては、時間を送信するサイトのブックマークがあります。 **ログ時間**が予約済みのキーワードの場合、組織内のユーザーが**ログ時刻**を検索すると、そのサイトは Microsoft 検索ボックスに唯一のブックマークとして表示されます。 

### <a name="group-related-content-with-keywords"></a>キーワードを含む関連コンテンツをグループ化する
特定の用語を検索するときに関連するコンテンツのセットをユーザーが検索できるようにするには、関連するすべてのコンテンツに同じキーワードを割り当てます。 例としては、ライフ状態の変化に関するプロセスとツールの検索があります。 更新の特典、税金情報、および名前とエイリアスの変更に対して回答をグループ化するには、 **marriage**のようなキーワードを含めます。

### <a name="search-settings"></a>検索の設定
検索設定を使用すると、コンテンツを調整して、特定のユーザーグループを対象にすることができます。 検索結果が表示されたときにこれらの Microsoft 検索設定を制御するには、次の操作を行います。

- **Date** 値です。 コンテンツがいつ利用可能かどうかを制御するには、開始日と終了日を設定します。 たとえば、時間に依存するマテリアルは、関連性がある場合に検索結果に表示されます。
- **国または地域**。 国または地域を選択できます。そのため、これらの場所のユーザーのみが特定のコンテンツを表示できます。 たとえば、国固有の情報は、それらの国の検索結果にのみ表示されます。
- **グループ**設定では、選択したグループのメンバーのみが結果を使用できるようになります。 たとえば、人事部の従業員にのみ関連するサイトを作成する場合は、この設定を適切な HR セキュリティグループにマップします。
- **デバイスまたは OS**。 デバイスの種類またはオペレーティングシステムを選択します。そのため、これらのデバイスを検索するユーザーまたはそれらのシステムを使用するユーザーのみがブックマークを表示します。
- **対象**指定したバリエーション。 この設定では、ユーザーのデバイスと場所に基づいて、ブックマークの内容が変化します。

## <a name="test-your-content"></a>コンテンツをテストする
[ブックマーク](manage-bookmarks.md)および[Q&を](manage-qas.md)作成したら、次の結果を確認します。
- 正しいブックマークまたは Q&A が表示されます。
- キーワードと一緒にグループ化されたすべてのコンテンツは、計画どおりに表示されます。
- 検索応答に予期しない場合があります。
- ブックマークまたは Q&には、十分な情報が含まれています。

コンテンツ作成に寄与するユーザーと Sme は、検索結果のテストと検証に役立ちます。

## <a name="review-and-update-periodically"></a>定期的に確認と更新を行う
[ブックマーク](manage-bookmarks.md)や[Q&の権限のある情報は、最新の](manage-qas.md)状態に保つ必要があります。 次の手順を定期的に実行します。
- 壊れた Url と無効な Url を修正または削除します。
- 不要になったブックマークまたは Q&を削除します。
- ツール、サイト名、チーム名の変更を確認する。
- ブックマークまたは Q&A が権限を持っているかどうかを検討します。また、わかりやすい説明が必要な場合もあります。

## <a name="get-insights-about-bookmarks-qa-and-locations"></a>ブックマーク、Q&A、および場所に関する洞察を得る

Microsoft Search では、[ブックマーク](manage-bookmarks.md)、 [Q&A](manage-qas.md)、および[場所](manage-locations.md)が発行、スケジュール、または提案される数を示します。 [Insights ダッシュボード](get-insights.md)には、状態別のブックマーク、Q&A、および場所の合計が表示されます。

- **公開済み:** ユーザーが利用可能な公開済みの結果の数。
- **スケジュール済み:** 発行パイプラインのスケジュール済みの結果の数。
- **おすすめ:** ユーザーからのおすすめの数。

推奨される[ブックマーク](manage-bookmarks.md)、 [Q&A](manage-qas.md)、および[場所](manage-locations.md)は、コンテンツ内のギャップの指標として適しています。 ユーザーが探しているのに見つからないことを理解するのに役立ちます。 このデータは、さらに多くのブックマーク、Q&A、または場所を作成する必要があることを示しています。 または、コンテンツをより見つけやすくするために、より良いキーワード、予約済みキーワード、および検索文字列を使用して、既存のコンテンツを更新する必要がある場合があります。

### <a name="review-top-search-queries"></a>よく使用される検索クエリを確認する

過去90日間にどの検索が最もインプレッションを生成したかを調べるには、上位の検索クエリを確認します。 *インプレッション*は、検索結果にページが表示された回数を表します。 [Insights ダッシュボード](get-insights.md)の**上位のクエリ**カードは、各検索結果の種類、検索の合計数、およびクリックスルーレート (CTR) を検索する上位25ユーザーを示しています。 このレポートを使用すると、検索クエリのボリュームを識別して、検索アクティビティが高および低のクエリを決定できます。

低探索数は、ユーザーの不満を示している可能性があります。 ユーザーがコンテンツを探していないか、または別のキーワードを使用して検索しています。 CTR は、ユーザーが昇格された結果を選択する頻度、およびユーザーにとってのクエリのルールと結果の有用性を示します。 Low CTR は、ユーザーがコンテンツを見つけたが、自分のニーズを満たしていないことを示します。 そのような場合は、コンテンツを確認してください。 コンテンツを検索クエリに配置するには、ユーザーの検索に一致することを確認し、タイトル、説明、およびキーワードを更新します。 

### <a name="analyze-impressions-by-result-type"></a>結果の種類別にインプレッションを分析する

[洞察ダッシュボード](get-insights.md)の**インプレッション配布**カードに表示される、さまざまな時間枠のインプレッションを表示します。 タイムラインでは、1 つの結果の種類に関して 1 日のインプレッション数が表示されます。 これらのグラフを使用すると、最も頻繁に使用される、またはあまり使用されない検索結果の種類を判断できます。 特定の結果の種類を不定期に使用することは、結果の種類が適切ではないことを意味するわけではありません。 単に、ユーザーが検索結果をどのように使用しているかを示しているだけです。

 特定の結果の種類がユーザーに推奨されている場合は、同じ種類の検索結果をさらに作成できます。 キーワードが適切であることを確認するには、使用率の低い結果の種類のキーワードを確認します。 このレポートを使用すると、時間の経過に伴うユーザーの動作の変更を確認することもできます。
