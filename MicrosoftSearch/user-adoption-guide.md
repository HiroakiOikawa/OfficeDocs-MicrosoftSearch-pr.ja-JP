# <a name="user-adoption-guide"></a>ユーザーの導入ガイド

この情報は、Microsoft Search 管理者、変更管理部門のリーダー、およびビジネスの所有者です。
  
展開し、組織全体にわたって Microsoft の検索の導入の成功には、この 4 段階のプロセスを推奨します。
  
## <a name="stage-1-stakeholders"></a>段階 1: 関係者

シームレスな導入を確保し、Microsoft Search のユーザーが積極的にサポートおよび促進するプロセスを支援できる人を特定します。
  
チームの階層構造と各ユーザーの貢献を表に示します。 
  
 **ユーザーの導入チームの役割と責任**
  
|**ロール**|**責任**|**部署**|
|:-----|:-----|:-----|
|エグゼクティブ スポンサー  <br/> |高度なビジョンと、組織の Microsoft の検索の値を理解します。  <br/> リーダーシップのサポートを提供する IT/ビジネス ・ チーム  <br/> リソースを割り当てください。  <br/> 従業員と直接通信します。  <br/> |経営幹部のリーダーシップ  <br/> |
|ビジネス オーナー  <br/> |製品とユーザーのシナリオを特定します。  <br/> 部門や組織の成功判断基準を決定します。  <br/> |ANY  <br/> |
|変更管理チーム  <br/> |製品の価値がわかりやすいことを確認します。  <br/> 公開の計画を作成します。  <br/> |IT  <br/> |
|エキスパート  <br/> |同僚およびチームに製品を宣伝します。  <br/> |(部下)  <br/> |
|検索管理者  <br/> |ユーザーの採用のコア チームのメンバーを識別します。  <br/> 経験と継続的な管理の外観と動作を含む、Microsoft の検索を管理します。  <br/> Microsoft Search の値を表示するのにはドライブのディスカッション  <br/> |IT  <br/> |
|エディター  <br/> |管理ポータルでのコンテンツを作成します。  <br/> |IT/任意  <br/> |
   
## <a name="stage-2-scenarios"></a>段階 2: シナリオ

ユーザーの採用を確実に判断し、エンドユーザーのシナリオを定義するが重要です。人が Microsoft Search を使用して、質問に答えるし、タスクの実行時間が短く、必要な情報を検索する方法の実際の例が含まれます。ロール アウトの成功を測定するためにユーザーの受け入れも開始する前に使用する測定基準を確認しておくこととします。
  
 **ユーザーが必要し、目的を理解します。**
  
に関しての情報を検索し、回答を得るには、ユーザーのニーズや問題点を把握するが重要です。定量的方法、Office 365 と SharePoint の検索の利用状況、または定性的な方法からのデータと同様に、調査し、このデータを取得するユーザーのフォーカス グループのようなを使用することができます。こう場合、Microsoft Search が価値を提供できる場所、および方法を決定します。
  
さまざまな役割や、組織全体でのチームは、次のように検討します。
  
- 既存ユーザーのペイン ・ ポイントの情報を検索する場合。
    
- 頻繁にアプリケーション、ツール、サイト、およびその他の情報ソースを使用します。
    
- どのようなユーザーにはに関しての情報を検索するのにを参照してください。
    
- どのように Microsoft Search の利点は、組織に適用されます。詳細については、 [Microsoft Search 理由](why-microsoft-search.md)を参照してください。
    
ユーザーの最大の正の影響を与えるユーザーのシナリオを選択します。
  
 **成功判断基準を特定します。**
  
ロールアウトの目標を達成して、組織にプラスの効果を表示するために、することが重要、重要な成功判断基準の決定を開始する前にします。シンプルかつ測定可能な目標を維持し、定期的にパフォーマンスを評価することが迅速に対応が必要な場合。
  
1. **変更管理:** Microsoft Search の認識との通信が成功したをどのように測定されますを決定します。 
    
2. **使用法:** データを活用することができます (DAU と MAU) 毎日および毎月のアクティブ ユーザーを含む分析ダッシュ ボードにコンテンツ タイプ、ビューでは、最もを取得しているし、トップを時間の経過とともに利用状況を測定するためのキーワードを検索します。詳細については、[インサイトのダッシュ ボードの使用](get-insights.md)を参照してください。
    
3. **ユーザーの満足度:** アンケートや、ベースラインを決定するのにはフォーカス グループのユーザーから収集したデータを使用します。Net の満足度 (NSAT) とネット プロモーター スコア (NPS) では、便利な測定値も指定できます。どの方法を使用して、一貫性を維持する時間の経過と共に影響を確認するためです。 
    
4. **その他の測定値:** 時間の検索のサポート ・ リクエストの履歴や現在のデータを使用します。 
    
## <a name="stage-3-readiness"></a>段階 3: 準備

Microsoft Search ロールアウトの準備に時間を費やすことと、組織にプラスの影響が発生します。 
  
 **製品とコンテンツの準備**
  
直接、管理者は、エンド ・ ユーザーの検索の操作性に影響を与えるし、組織の Microsoft Search の外観を決定します。これには、ユーザーに公開する結果の種類を選択することが含まれます。点灯と消灯のドキュメント、サイト、ユーザー、グループ、または会話の結果では、これを制御します。
  
詳細については、 [Microsoft の検索の設定](set-up-microsoft-search.md)を参照してください。
  
エディターは、それぞれの専門分野内のブックマークと Q&As の管理を支援する特定分野のエキスパートです。エディターには、以前識別したユーザーのシナリオに対応するコンテンツを作成する必要があります。
  
詳細については、[コンテンツの計画](plan-your-content.md)を参照してください。
  
 **サインイン**
  
Microsoft Search のユーザーは、職場または学校のアカウントを使用して Bing にサインインしなければなりません。サインイン可能な限りシームレスのユーザー。
  
- Azure Active directory 構成が正しく設定されている、少数のユーザーがサインインしようとすることを確認します。詳細については、[テストのシングル サインオン](test-single-sign-on.md)を参照してください。
    
- Bing の会社で管理されるデバイスのブラウザーの既定値を設定します。詳細については、[既定のブラウザーの設定](set-default-browser.md)を参照してください。
    
- グループ ポリシーを使用してエッジとクロムのユーザーの検索の拡張機能を展開します。詳細については、[管理ポータルのツール](admin-portal-tools.md)を参照してください。
    
 **ドラフト通信**
  
Microsoft Search と時間の節約に役立つ方法についてユーザーに通知するのには、現実世界のシナリオを使用します。変更管理チームは、展開戦略を作成し、エンド ・ ユーザーの準備のための関連資料とキャンペーンの準備に役立ちます。
  
通信と、組織のイベントの計画を開始します。
  
- 横断幕、ポスター、または leaflets は、目立つ場所に配置
    
- 人で製品のデモまたはブース
    
- ローカルおよびリモートのチームのオンライン イベント
    
- エグゼクティブ スポンサー、町役場の会議をホストしているか、お知らせメールを送信
    
- Microsoft Search を使用してのセルフ ヘルプ ドキュメントを共有します。
    
すべての通信には、方法にサインインできる Bing の作業の結果を検索するについてユーザーを教育する必要があります。
  
 **パートナー ・ チームを関与します。**
  
Microsoft 365 の不可欠な部分とは、Microsoft Search は、GDPR と層 C のガイドラインに準拠します。組織によっては、プライバシーおよびセキュリティ チームとマイクロソフトの検索に関する情報を共有するために必要な場合があります。詳細については、 [Microsoft の検索のためのセキュリティ](security.md)を参照してください。
  
ユーザーをより効果的にサポートできるように、機能と Microsoft の検索の機能だけでなくにサインインするユーザーの要件について、ヘルプ デスクが知っていることを確認します。詳細については、 [Microsoft Search を使用](use/about-microsoft-search.md)し、 [Faq](faqs.md)を参照してください。
  
 **パイロット ユーザーと評価します。**
  
Microsoft Search の影響を評価し、早期のフィードバックを収集するユーザーの小さなグループから開始します。このようにする、コンテンツを変更するのには、通信、および広範なロールアウトの期待値を設定を更新します。パワー ユーザーの検索や初期採用者は、この優れた候補です。
  
パイロット ユーザーを検索するには。
  
- 組織内でのビジネス ・ スポンサーとの提携します。
    
- 早期採用者のグループを識別します。
    
- 大量のエンタープライズ検索を実行するグループを特定します。
    
- 国際など、他の地理的な場所でユーザーを識別します。
    
パイロット フィードバックのオプションを使用して Microsoft の検索や簡単なアンケートに入力するユーザーに問い合わせてください。
  
アンケートの点を確認します。 
  
- サインインして、Microsoft Search を使用する方法を理解するでしょうか。
    
- 期待どおりですか。
    
- 機能の操作を行いますか?
    
- 作業の結果を見つけることができましたか。
    
- 作業の結果は役に立つでしたか。
    
- Microsoft Search が明確で分かりやすかった通信されます。
    
これは、ステージ 2 で特定された指標の有効性の測定を開始する絶好の機会です。
  
パイロット フェーズを開始するときより広範な組織で Microsoft Search のプレビューを共有を検討してください。
  
## <a name="stage-4-roll-out"></a>段階 4: ロール アウト

すべての組織は、製品展開の方法が異なると考えています。いくつかの考慮事項を次に示します。
  
 **幅広い展開**
  
準備ができたら、access には、組織、Microsoft Search では、すべてのユーザーを許可するか、または段階的なアプローチを使用して、ゆっくりと詳細を展開します。 
  
 **意識を生成します。**
  
Microsoft Search は、組織内の興奮を構築します。
  
- ピア ツー ピアの支援および専門知識を提供するパイロット ユーザーを参加させます。
    
- 通信とステージ 3 で計画したイベントに従ってください。
    
 **継続的なコミュニケーション**
  
最初のいくつかの週および月、マイクロソフトの検索との連携を促進するコミュニケーションのリズムを維持します。これは、ユーザーの通知を維持、新規ユーザーが関与します。各通信は、Microsoft Search を自分の仕事に役に立ちます理由説明必要があります。
  
ユーザーの成功事例と同様に機能と検索には、強調表示する進行中の通信を送信します。
  
 **フィードバックを収集し、向上させる**
  
ユーザーをサポートしでそれらを聞くと、フィードバックに作用するように注意するかを知らせます。
  
分析ダッシュ ボードを使用すると、製品およびコンテンツの実行方法を参照してください。ブックマーク、Q&As、およびその他の通信を更新するには、この情報を活用します。これも役立ちます。
  
1. コンテンツの品質を向上させる: レビューのタイトル、説明、および Url、キーワードの最適化、古い情報を削除、新しい情報を追加
    
2. 機能は、最も頻繁に確認、または頻度の低い-使用
    
3. アンケート、フォーカス グループ、および Microsoft Search のユーザーの意見を理解して、採用を増加するフィードバック ・ セッションを実施し、改善分野を特定
    
教育とフィードバックは、反復的なプロセスです。Microsoft Search を採用するコンテンツや、ユーザーとの通信に変更を加える必要がある場合があります。
