# <a name="user-adoption-guide"></a>ユーザー導入ガイド

この情報は、Microsoft Search 管理者、変更管理リーダー、およびビジネス オーナーを対象としています。
  
組織全体に Microsoft Search をロールアウトし、適切な導入を確実に行うには、次に示す 4 段階の手順を実行することをお勧めします。
  
## <a name="stage-1-stakeholders"></a>ステージ 1: 関係者

シームレスな展開を確実にし、Microsoft Search のユーザーの導入を推進するために、プロセスをサポートし、推進できるユーザーを特定します。
  
次の表は、チームの階層と各ユーザーの貢献度を示しています。 
  
 **ユーザー導入チームの役割と責任範囲**
  
|**役割**|**責任範囲**|**部署**|
|:-----|:-----|:-----|
|エグゼクティブ スポンサー  <br/> |組織の Microsoft Search の高度なビジョンと価値を理解する  <br/> IT/ビジネス チームにリーダーシップのサポートを提供する  <br/> リソースを割り当てる  <br/> 従業員と直接連絡を取る  <br/> |経営幹部のリーダー  <br/> |
|ビジネス オーナー  <br/> |製品とユーザーのシナリオを特定する  <br/> 部署と組織の成功指標を決定する  <br/> |任意  <br/> |
|変更管理チーム  <br/> |製品価値の理解が容易であることを確認する  <br/> ロールアウトの計画を作成する  <br/> |IT  <br/> |
|支持者  <br/> |同僚やチームに製品を広める  <br/> |任意 (スタッフ)  <br/> |
|検索管理者  <br/> |ユーザー導入の中心チーム メンバーを特定する  <br/> エクスペリエンスと継続的な管理のルック アンド フィールを含む Microsoft Search を管理する  <br/> Microsoft Search の価値を示すディスカッションを推進する  <br/> |IT  <br/> |
|編集者  <br/> |管理ポータルでコンテンツを作成する  <br/> |IT/任意  <br/> |
   
## <a name="stage-2-scenarios"></a>ステージ 2: シナリオ

ユーザー導入を確実にするには、エンド ユーザーのシナリオを決定し定義することが重要です。 これらには、ユーザーが Microsoft Search を使用して必要な情報を検索し、質問に回答して、迅速にタスクを完了するための実例が含まれています。 ロールアウトとユーザー導入の成功を評価するために、開始する前に使用する指標を決定することも重要です。
  
 **ユーザーが必要とするものを理解する**
  
情報を検索して回答を得るにはユーザーのニーズや問題点が重要になることを理解する必要があります。 このデータを取得するには、量的な方法 (Office 365 や SharePoint 検索の使用状況などのデータ) や、質的な方法 (アンケートやユーザー フォーカス グループ) を使用できます。 これは、Microsoft Search の価値を提供できる状況、場所、方法を判断するのに役立ちます。
  
組織全体の役割とチームについては、次の点を考慮してください。
  
- 既存のユーザーが情報を検索する際の問題点。
    
- 使用頻度の高いアプリ、ツール、サイト、その他の情報ソース。
    
- ユーザーが情報を検索する際に表示される内容。
    
- Microsoft Search の利点を組織に適用する方法。 詳細については、「[Microsoft Search を使用する利点](why-microsoft-search.md)」を参照してください。
    
ユーザーに最大の好影響を与えるユーザー シナリオを選択します。
  
 **成功指標を特定する**
  
展開の目標を達成し、組織にとって好影響を示すために、開始する前に主要な成功指標を決定することが重要です。 目標をシンプルで評価可能なものにし、必要に応じて迅速に対応できるように、定期的に業績を評価します。
  
1. **変更管理:** Microsoft Search の認識を評価する方法と、それに関するコミュニケーションの成否を評価する方法を決定します。 
    
2. **使用状況:** Insights ダッシュボードのデータを活用できます。このデータには、毎日および毎月のアクティブなユーザー (dau および mau)、最も多くのビューを取得しているコンテンツタイプ、時間の経過を評価する上位の検索キーワードが含まれます。 詳細については、「[Insights ダッシュボードを使用する](get-insights.md)」を参照してください。
    
3. **ユーザーの満足度:** アンケートおよびユーザー フォーカス グループから収集したデータを使用して、基準を決定します。 満足度指標 (NSAT) とネット プロモーター スコア (NPS) も、有用な評価値を提供できます。 使用する方法に関係なく、一貫性を維持し、時間の経過による影響を確認できるようにします。 
    
4. **その他の評価値:** サポートの要求と検索にかかった時間に対して、過去または現在のデータを使用します。 
    
## <a name="stage-3-readiness"></a>ステージ 3: 準備

Microsoft Search のロールアウト準備に時間をかけることで、組織にとってのメリットが得られます。 
  
 **製品とコンテンツの準備**
  
管理者は、エンド ユーザーの検索機能に直接影響を及ぼし、Microsoft Search で組織を検索する方法を決定します。 これには、ユーザーに表示する結果の種類の選択が含まれます。 ドキュメント、サイト、人々、グループ、会話の結果を有効または無効にすることで、これを制御します。
  
詳細については、「[Microsoft Search をセットアップする](set-up-microsoft-search.md)」を参照してください。
  
編集者は、専門分野でブックマークと Q&A を管理するために役立つ専門家です。 編集者は、以前に確認したユーザー シナリオに対応するコンテンツを作成する必要があります。
  
詳細については、「[コンテンツを計画する](plan-your-content.md)」を参照してください。
  
 **サインイン**
  
Microsoft Search ユーザーは、職場または学校のアカウントを使用して Bing にサインインする必要があります。 ユーザーができるだけシームレスにサインインできるようにするには、次のようにします。
  
- Azure Active directory の構成が正しく設定されていることを確認し、少数のユーザーでサインインを試行します。 シングル サインオンの詳細については、「[シングル サインオンのテスト](test-single-sign-on.md)」を参照してください。
    
- 会社で管理されているデバイスの既定のブラウザーを Bing に設定します。 詳細については、「[既定のブラウザーを設定する](set-default-browser.md)」を参照してください。
    
- グループ ポリシーを使用して、エッジおよび Chrome ユーザーの検索拡張機能を展開します。 詳細については、「[管理ポータルのツール](admin-portal-tools.md)」を参照してください。
    
 **コミュニケーションの原稿**
  
実際のシナリオを使用して、ユーザーに Microsoft Search についてと、どのように時間を節約できるかを説明します。 変更管理チームは、ロールアウト戦略を作成し、エンドユーザーの準備状況に向けた販促用品やキャンペーンの準備を支援します。
  
組織のコミュニケーションとイベントの計画を開始します。
  
- 目立つ場所に配置されたバナー、ポスター、リーフレット
    
- 個人向けの製品デモまたはブース
    
- ローカル チームとリモート チームのオンライン イベント
    
- エグゼクティブ スポンサーがタウン ホール ミーティングをホストしたり、アナウンス メールを送信する
    
- Microsoft Search の使用に関するセルフヘルプ ドキュメントを共有する
    
すべてのコミュニケーションで、Bing にサインインして作業結果を検索する方法について、ユーザーを教育する必要があります。
  
 **パートナー チームと連携する**
  
Microsoft 365 の重要な部分として、Microsoft Search は GDPR および Tier C のガイドラインに準拠しています。 組織によっては、プライバシーおよびセキュリティ チームと Microsoft Search に関する情報を共有する必要がある場合があります。 詳細については、「[Microsoft Search のセキュリティ](security.md)」を参照してください。
  
ヘルプデスクが Microsoft Search の機能とサインインするユーザー要件について理解していることを確認して、より効果的にユーザーをサポートできるようにしてください。 詳細については、「[Microsoft Search を使用する](use/about-microsoft-search.md)」と「[よくある質問](faqs.md)」を参照してください。
  
 **パイロット ユーザーで評価する**
  
少数のユーザー グループを使用して、Microsoft Search の影響を評価し、早期のフィードバックを収集します。 これにより、コンテンツを変更したり、コミュニケーションを更新したり、広範なロールアウトに対する期待値を設定することができます。 最適な候補として、検索のパワー ユーザーや初期導入者があります。
  
パイロット ユーザーの検索を支援するには、次のようにします。
  
- 組織内のビジネス スポンサーと協力する
    
- 早期導入者のグループを特定する
    
- 大量のエンタープライズ検索を実行するグループを特定する
    
- 海外などの他の地域にいるユーザーを特定する
    
Microsoft Search のフィードバック オプションまたは簡単なアンケートを使用して、パイロット ユーザーに入力を依頼します。
  
次の質問に対するアンケートをします。 
  
- Microsoft Search にサインインして使用する方法を理解していますか。
    
- 期待どおりに動作していますか。
    
- お気に入りの機能はどれですか。
    
- 作業の結果を見つけることができましたか。
    
- 作業の結果は役に立ちましたか。
    
- Microsoft Search に関するコミュニケーションは明確で理解できるものですか。
    
これは、ステージ 2 で特定された指標の有効性を評価する絶好の機会です。
  
パイロット フェーズを開始するときは、Microsoft Search のヒントを広い組織と共有することを検討してください。
  
## <a name="stage-4-roll-out"></a>ステージ 4: ロール アウト

どの組織も製品の展開について異なる考え方をしています。 ここでは、いくつかの考え方を説明します。
  
 **幅広い展開**
  
準備が整ったら、組織内のすべてのユーザーに Microsoft Search へのアクセスを許可するか、段階的なアプローチを使用してゆっくりロール アウトします。 
  
 **意識を高める**
  
Microsoft Search について組織内の関心を盛り上げるには、次のようにします。
  
- パイロット ユーザーを参加させて、ピアツーピアのサポートや専門知識を提供します。
    
- ステージ 3 で計画したコミュニケーションとイベントをフォローします。
    
 **継続的なコミュニケーション**
  
最初の数週間および数か月はコミュニケーションのリズムを管理して、Microsoft Search による活動を促進します。 これにより、ユーザーは常に情報を入手したり、新しいユーザーを得ることができます。 各コミュニケーションは、Microsoft Search が自分の仕事に役立つ理由を説明する必要があります。
  
機能および検索を強調している継続的なコミュニケーションを、ユーザーの成功事例と共に送信します。
  
 **フィードバックを収集して改善する**
  
ユーザーをサポートし、意見を聞き、ユーザーのフィードバックに基づいて行動します。
  
製品とコンテンツがどのように実行されているかを確認するには、Insights ダッシュボードを使用します。 この情報を活用して、ブックマーク、Q&A、およびその他のコミュニケーションを更新します。 これは、次のような場合にも役立ちます。
  
1. コンテンツの品質の向上: タイトル、説明、URL の確認、キーワードの最適化、古くなった情報の削除、新しい情報の追加を行います
    
2. 最も頻繁に使用される機能や、あまり使用されない機能を特定します
    
3. アンケート、フォーカス グループ、フィードバック セッションを実施して、ユーザーが Microsoft Search についてどう考えているかを理解し、導入を促進し、改善範囲を特定します
    
教育とフィードバックは反復的なプロセスです。 ユーザーが Microsoft Search を導入する際には、コンテンツやコミュニケーションの変更が必要になる場合があります。
